1. 需求阶段
你提出想要一个能根据《暗区突围无限》不同模式（普通/封锁/强化封锁）随机生成符合战备值要求的装备的工具。核心难点在于：
实时价格获取：由于游戏市场价格波动大，我们决定采用静态价格表，由你后续根据游戏现状手动更新。
配件兼容性：不同武器平台的配件不能混用，必须为每把武器单独建立兼容列表。
2. 技术方案
我们选择用 Python 开发，利用 Claude Code 来辅助编码。主要文件：
equipment_data.py：存储所有装备的静态数据（名称、价格、配件兼容关系）。初期我们会填充常见装备的示例数据，后续你可以自行增改。
generator.py：核心随机生成逻辑。它会根据选择的模式（普通、封锁、强化封锁）确定战备值区间，然后从数据中随机组合，并通过循环调整使总价落在区间内。
main.py：简单的命令行交互，让用户选择模式，调用生成器并打印结果。
3. 实施步骤（你将与 Claude Code 协作完成）
第一步：搭建骨架
在 Claude Code 中，你将我上面提供的提示词粘贴进去，它会自动生成三个文件的基础代码，并填入合理的示例数据（例如头盔价格、AKM 配件等）。
第二步：数据修正与补充
价格：查看生成的静态价格是否合理，若偏离较大，直接告诉 Claude Code：“把 AKM 价格改为 18000，M4A1 改为 22000，调整相应配件价格。”x它会自动修改。
配件兼容性：对于 H416、SCAR-L、MPX 等武器，你需要确保配件列表正确。可以让 Claude Code 搜索网络资料，或者你从游戏社区找数据后让它更新。如果它生成的配件有误，直接指出：“SCAR-L 不能用 AR 弹匣，请单独列出它的弹匣型号。”它会立即修正。
第三步：测试运行
在 Claude Code 中让它执行 python main.py，观察生成的装备是否合理。如果总价总是不达标，可以要求它调整生成策略（例如“优先选择接近目标价位的装备”）。
第四步：迭代优化
根据测试结果，你可以提出进一步需求：
“输出时用 rich 库美化表格。”
“增加一个功能：记录最近 10 次生成的组合，避免重复。”
“允许用户固定某个部位（如指定用‘哨兵护甲’）再随机其他。”
Claude Code 会逐步完善代码。
第五步：数据维护
由于采用静态价格，你需要定期更新装备价格。可以直接在 equipment_data.py 中修改数值，或者让 Claude Code 帮你写一个简单的脚本，读取 CSV 表格来更新价格。
4. 最终成果
你将得到一个命令行工具，输入模式后即可获得一套合理配置的装备清单，方便你在游戏中快速准备战备。后续你还可以扩展图形界面、增加装备评分等功能。
